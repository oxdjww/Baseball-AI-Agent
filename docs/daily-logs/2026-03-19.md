# 2026-03-19 개발 로그

## 주요 작업

### 1. 경기 종료 알림 순위 정확도 개선 — StandingsAdjuster 도입

**문제**: 경기 종료 직후 `GameProcessor`가 `NaverStandingsClient.fetchStandings()`를 호출하지만,
Naver 순위 API는 방금 끝난 경기 결과를 즉시 반영하지 않아 팬에게 **이전 순위(stale data)** 가 전송됨.

**해결 방향**: 외부 API 재시도나 지연 발송 없이, 이미 알고 있는 경기 결과를 순위표에 직접 적용.

**핵심 아이디어**:
```
(Naver 순위표: 아직 미반영) + (RealtimeGameInfoDto: 방금 끝난 결과 포함)
         ↓  StandingsAdjuster.applyGameResult()
    (보정된 순위표: 금일 경기 결과 반영)
         ↓
    GameEndNotificationBuilder.build() → TelegramService
```

#### StandingsAdjuster 동작

1. **결과 적용**: winner → `wins + 1`, loser → `losses + 1`, 무승부 → 양 팀 `draws + 1`
2. **재정렬**: 승률(`wins / (wins+losses)`) 내림차순 → 승수 내림차순 (KBO 동률 기준)
3. **rank 재부여**: 1-인덱스 기반 재배정
4. **승차 재계산**: `GB = ((1위승 - 팀승) + (팀패 - 1위패)) / 2` (KBO 표준 공식)

#### 엣지 케이스 처리

| 상황 | 처리 |
|---|---|
| `rawStandings == null` | null 반환 (기존 "조회 실패" 처리 흐름 유지) |
| 경기 취소 (`isCanceled = true`) | 원본 반환 (stats 변경 없음) |
| 무승부 | 양 팀 `draws + 1` |
| 팀 코드 미매핑 | 원본 반환 + warn 로그 |
| 시즌 초 0경기 팀 | `Math.max(1, wins+losses)` — 0 나누기 방지 |
| 1위 팀 GB 부동소수점 오차 | `Math.max(0.0, gb)` — `-0.0` 방지 |
