# 2026-03-29 개발 로그

## 주요 작업

### 1. Naver 순위 API teamName 기반 팀 코드 매핑 수정

#### 문제

경기 종료 알림 발송 시 순위 정보 조회 단계에서 `[teamCode 'LG' 매핑 없음]` 오류 발생.

Naver 순위 API가 반환하는 `teamId` 값이 내부 `Team` enum 코드와 일치하지 않아 매핑 실패.

#### 원인

`NaverStandingsClient`가 API 응답의 `teamId` 필드로 `Team` enum을 조회하고 있었으나,
Naver API는 `teamId`에 풀네임(`"LG트윈스"` 등)을 담아 내려보냄.

#### 수정

| 유형 | 내용 |
|------|------|
| 수정 | `Team` enum에 `naverTeamName` 필드 추가 (`"LG트윈스"` 등 Naver 풀네임) |
| 신규 | `Team.fromNaverTeamName()` 정적 메서드 — 풀네임 → 내부 코드 변환 |
| 수정 | `NaverStandingsClient`: `teamId` 기반 변환 → `teamName` 기반 변환으로 교체 |

#### 변경 파일

| 파일 | 내용 |
|------|------|
| `domain/team/Team.java` | `naverTeamName` 필드 + `fromNaverTeamName()` 추가 |
| `external/naver/NaverStandingsClient.java` | `teamName` 기반 매핑으로 교체 |
