### 2. Member 복합키 → 단일키 리팩터링 + telegram_id 중복 가입 버그 수정

#### 버그 원인

`Member` 테이블 PK가 `(id, telegram_id)` 복합키로 설계되어 있었으나, `id`가 `SEQUENCE` 자동 생성이므로
동일한 `telegram_id`를 가진 행도 항상 다른 PK 쌍을 가지게 되어 중복 insert가 허용되었음.

```
PK = (id, telegram_id)
      ↑
      AUTO-GENERATED (항상 유일) → 중복 telegram_id 차단 불가
```

#### 수정 내용

- 복합키 구조(`@IdClass`, `Persistable<MemberId>`, `MemberId.java`) 전체 제거
- 단일 PK(`id`) 복원 + `@Column(unique = true)` on `telegramId`
- DB 마이그레이션: `V3__fix_member_pk_and_unique_telegram_id.sql`
  - `ALTER TABLE member DROP CONSTRAINT member_pkey`
  - `ALTER TABLE member ADD CONSTRAINT member_pkey PRIMARY KEY (id)`
  - `ALTER TABLE member ADD CONSTRAINT member_telegram_id_unique UNIQUE (telegram_id)`
- `getSeqId()` → `getId()` 전체 교체 (MemberService, HomeController, admin 템플릿 포함)

#### 변경 파일

| 유형 | 파일 |
|---|---|
| 삭제 | `member/MemberId.java` |
| 수정 | `member/Member.java` — 단일 PK, unique 컬럼 |
| 수정 | `member/MemberRepository.java` — `JpaRepository<Member, Long>` |
| 수정 | `member/MemberService.java` — `getSeqId()` → `getId()` |
| 수정 | `web/HomeController.java` — `getSeqId()` → `getId()` |
| 수정 | `templates/admin/member.html` — `m.seqId` → `m.id` |
| 수정 | `templates/admin/member_form.html` — `member.seqId` → `member.id` |
| 신규 | `db/init/V3__fix_member_pk_and_unique_telegram_id.sql` |
| 수정 | `test/.../MemberSingleKeyTest.java` — 단일키 + 중복 저장 시 예외 검증 테스트 |

> **Production 조치**: V3 마이그레이션을 운영 DB에 직접 적용 (init 스크립트는 최초 생성 시에만 실행되므로 수동 적용 필요).
> 적용 전 중복 행 4개 정리 (telegram_id=7784842933, id=23/25/26/27 삭제, id=3 보존).

### 6. 중복 telegram_id 가입 시도 시 사용자 안내 메시지

#### 문제

DB UNIQUE 제약은 정상 동작하지만 `DataIntegrityViolationException`이 `@Transactional` 밖으로 전파되어
Telegram 사용자에게 아무 안내 없이 500 에러 처리.

#### 수정

save 직전 `existsByTelegramId()` 사전 체크 → `LinkResult.AlreadyLinked` 반환 → 안내 메시지 전송.

```
existsByTelegramId() == true
    ↓
LinkResult.AlreadyLinked 반환
    ↓
"이미 가입된 텔레그램 계정입니다. 기존 계정으로 로그인해주세요 👇 <로그인 URL>"
```

#### 변경 파일

| 유형 | 파일 |
|---|---|
| 수정 | `auth/LinkResult.java` — `AlreadyLinked` 케이스 추가 |
| 수정 | `auth/TelegramLinkService.java` — save 전 중복 체크 |
| 수정 | `member/MemberRepository.java` — `existsByTelegramId()` 추가 |
| 수정 | `notification/telegram/TelegramWebhookService.java` — AlreadyLinked 처리 |
| 수정 | `template/NotificationTemplate.java` — `ALREADY_LINKED` 메시지 추가 |
