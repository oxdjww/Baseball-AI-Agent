### 3. 회원탈퇴(soft delete) 기능 추가

#### 설계

물리 삭제 대신 `deleted` 컬럼 기반 soft delete 채택.
`@SQLRestriction("deleted = false")` 를 Entity에 선언해 탈퇴 회원이 알림 발송·로그인 쿼리에서 자동 제외되도록 처리.

#### 재가입 허용을 위한 Partial Unique Index

탈퇴 후 같은 Telegram 계정으로 재가입 시 기존 `UNIQUE` 제약이 INSERT를 막는 문제 발생.
전체 unique 제약 → **Partial Unique Index**(`WHERE deleted = false`)로 교체해 탈퇴 행의 `telegram_id`는 유니크 검사에서 제외.

```sql
-- V4
ALTER TABLE member ADD COLUMN deleted BOOLEAN NOT NULL DEFAULT FALSE;

-- V5
ALTER TABLE member DROP CONSTRAINT member_telegram_id_unique;
CREATE UNIQUE INDEX member_telegram_id_active_unique
    ON member(telegram_id) WHERE deleted = false;
```

운영 DB에 직접 적용 완료. 기존 유저 2명 `deleted = false` 자동 설정 확인.

#### 변경 파일

| 유형 | 파일 |
|------|------|
| 수정 | `member/Member.java` — `deleted` 필드 + `@SQLRestriction` |
| 수정 | `member/MemberRepository.java` — `softDeleteById()` 추가 |
| 수정 | `web/HomeController.java` — `POST /home/withdraw` 엔드포인트 |
| 수정 | `templates/user/preferences.html` — 탈퇴 버튼 + `confirm()` 팝업 |

| 신규 | `db/init/V4__add_member_soft_delete.sql` |

### 4. Thymeleaf `th:replace` 구조적 버그 2회 반복 — 교훈

#### 문제

`preferences.html` 에서 `<body th:replace="~{fragments/layout :: layout(~{::section}, false)}">` 구조 특성상
`<section>` 태그 **밖**에 선언한 요소(모달 div, script 태그)가 Thymeleaf 렌더링 시 조용히 버려짐.

이 버그로 인해 두 가지 오류가 연속 발생:
1. Bootstrap 모달 HTML이 DOM에 없어 `Cannot read properties of undefined (reading 'backdrop')` JS 오류
2. `confirm()` 스크립트가 없어 탈퇴 팝업 미작동

#### 수정

모달 제거 → `confirm()` 팝업으로 단순화, `<script>` 태그를 `</section>` 닫기 전으로 이동.

> **재발 방지**: 이 레이아웃 패턴에서는 section 밖에 선언한 모든 요소(script, div 등)가 버려짐. 항상 section 안에 위치시킬 것.
