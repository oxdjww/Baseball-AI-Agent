# 2026-03-30 개발 로그

## 주요 작업

경기 없는 날 기존 로직 점검 — 코드 리뷰를 통해 버그 4건 수정 + 관제 알림 기반 Telegram 봇 차단 처리 기능 추가.

---

### 1. GameProcessor — API 재시도 소진 시 NPE 방어

#### 문제

`NaverSportsClient.fetchGameInfo()`는 `@Retryable`로 감싸져 있으며,
재시도 소진 시 `@Recover` 메서드(`recoverGameInfo`)가 예외 없이 `null`을 반환.

`GameProcessor`의 try-catch는 예외만 잡으므로 `null` 반환 케이스를 통과 →
이후 `info.getStatusCode()` 호출 시 NPE 발생.

#### 수정

try-catch 블록 직후 `null` 체크 추가 후 즉시 return.

| 파일 | 내용 |
|------|------|
| `game/alert/GameProcessor.java` | `info == null` 체크 + warn 로그 후 return |

### 5. Telegram 봇 차단(403) 자동 알림 비활성화

#### 배경

관제 로그에서 봇을 차단한 사용자에게 계속 메시지 전송을 시도해 403 에러가 반복 발생.
기존 코드는 모든 예외를 `catch (Exception e)`로 삼키고 로그만 기록 → 무의미한 API 호출 + 관제 노이즈 지속.

#### 수정

403 "bot was blocked by the user" 감지 시 해당 사용자의 알림 플래그 3개를 DB에서 자동 비활성화.

| 파일 | 내용 |
|------|------|
| `notification/telegram/BotBlockedException.java` | chatId 보존 RuntimeException 신규 생성 |
| `notification/telegram/TelegramNotificationClient.java` | `HttpClientErrorException` 분리 catch, 403+blocked 감지 시 `BotBlockedException` throw |
| `member/MemberRepository.java` | `disableAllNotificationsByTelegramId()` JPQL UPDATE 추가 |
| `notification/telegram/TelegramService.java` | `sendPersonalMessage()`에서 catch 후 알림 플래그 전체 비활성화 |

#### 처리 흐름

```
sendPersonalMessage(chatId, name, rawText)
  → telegramClient.sendMessage(msg)
      → 403 + "bot was blocked" 감지 → BotBlockedException throw
  ← catch(BotBlockedException)
      → memberRepository.disableAllNotificationsByTelegramId(chatId)
          → notifyGameAnalysis / notifyRainAlert / notifyRealTimeAlert = false
```

이후 알림 대상 쿼리(`findBySupportTeamAndNotifyRainAlertTrue` 등)에서 자동 제외.
`sendMessageToTeam()` / `sendAnnouncementToAllMembers()`는 내부적으로 `sendPersonalMessage()`를 호출하므로 추가 수정 없이 커버됨.
