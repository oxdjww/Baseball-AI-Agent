### 1. 선취점 알림 기능 완성

#### 배경

`LeadChangeNotifier`는 Redis에 이전 리더(`rawPrev`)를 저장해 역전 여부를 감지한다.
기존 코드는 `rawPrev == null`(최초 폴링)을 `prevLeader = "NONE"`으로 처리 →
경기 시작 후 첫 득점도 "역전"으로 인식하거나 무시하는 문제.

#### 수정

`rawPrev == null` 분기를 독립 처리: 선취점 감지 시 `formatFirstScore()`로 별도 메시지 발송.

| 파일 | 내용 |
|------|------|
| `game/alert/LeadChangeNotifier.java` | `rawPrev == null` 시 선취점 알림 분기 + try-catch 추가 |
| `game/message/GameMessageFormatter.java` | `formatFirstScore()` 신규 메서드 (내 팀/상대팀 선취 메시지 분기) |

#### 선취점 알림 흐름

```
rawPrev == null (최초 폴링)
  └─ currLeader != "NONE" → 선취점 감지
       → formatFirstScore(member, info, currLeader)
       → telegram.sendPersonalMessage(...)
  └─ currLeader == "NONE" (0:0) → 무시
```

### 5. 텔레그램 메시지 전송 이력 저장 기능 추가

#### 배경

메시지 발송 이력이 SLF4J 로그로만 남아 사후 조회 불가.
`telegram_send_log` 테이블을 신설해 모든 발송 시도를 DB에 적재.

#### 아키텍처 설계

```
TelegramService
  → TelegramNotificationClient  (HTTP 호출, TelegramSendResult 반환)
  → MessageLogService           (이력 저장, notification.log 패키지)
  → MemberRepository            (봇 차단 시 알림 비활성화)
```

- `MessageLogService`를 `notification.log` 패키지에 배치 →
  `MonitoringAspect`의 `!within(notification..)` 포인트컷에 의해 모니터링에서 자동 제외.
  `MessageLogService` 예외가 모니터링 텔레그램 알림을 재귀 호출하는 루프 방지.
- `REQUIRES_NEW` 트랜잭션 + 예외 swallow → 이력 저장 실패가 핵심 흐름 중단 안 함.

#### 수정

| 파일 | 내용 |
|------|------|
| `notification/telegram/dto/TelegramSendResult.java` | 전송 결과 값 객체 (record) |
| `notification/log/MessageSendLog.java` | 전송 이력 엔티티 |
| `notification/log/MessageSendLogRepository.java` | JPA Repository |
| `notification/log/MessageLogService.java` | 저장 전담 서비스 (REQUIRES_NEW) |
| `notification/telegram/TelegramNotificationClient.java` | void → TelegramSendResult 반환, BotBlockedException throw 제거 |
| `notification/telegram/TelegramService.java` | result 기반 봇 차단 분기 + messageLogService.save() 호출 |
| `notification/telegram/BotBlockedException.java` | @Deprecated 처리 |
| `db/init/V6__create_telegram_send_log.sql` | 참조용 DDL (실 DB는 직접 실행) |

#### 트러블슈팅: allocationSize 불일치

`BIGSERIAL`로 생성한 시퀀스(increment=1)에 `allocationSize=50` 설정 →
Hibernate 기동 시 `MappingException` 발생. `allocationSize=1`로 수정.
